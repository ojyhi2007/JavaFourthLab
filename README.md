## Задание 1.3 и 1.4  — Интерфейс сравнения и сравнение студентов 
<details>
<summary>Подробнее об задании</summary>
  
Интерфейс MyComparable и сравнение студентов

Описание:
Создан интерфейс MyComparable, который определяет общий способ сравнения объектов. На основе этого интерфейса класс Student реализует возможность сравнивать студентов между собой по их среднему баллу.

Логика решения:
Создан обобщённый интерфейс `MyComparable<T>`, содержащий метод compare(T other).
Метод возвращает целое число, показывающее результат сравнения объектов.

Класс Student реализует интерфейс `MyComparable<Student>`.
Сравнение выполняется на основе среднего балла студентов.

Метод compare() использует метод getAverageGrade() для вычисления среднего значения оценок и сравнивает два значения с помощью Double.compare().

Если средний балл текущего студента больше — метод возвращает положительное число.
Если средние баллы равны — возвращается 0.
Если средний балл меньше — возвращается отрицательное число.

Пример:
Создаются два студента:
Вася с оценками 3,4,5,4 и Петя с оценками 5,5,5,5.

## Задание 2.3 - Обобщённая коробка (создание класса `Box<T>` для хранения объектов)
<details>
<summary>Подробнее об задании</summary>
  
Обобщённая коробка `Box<T>`

Описание:   
Создан обобщённый класс `Box<T>`, который предназначен для хранения одного объекта произвольного типа. Класс позволяет поместить объект в коробку, достать его из коробки и проверить, пуста ли коробка. Дополнительно реализована работа с трёхмерной точкой Point3D через wildcard.

Логика решения:   
Класс `Box<T>` содержит одно поле типа T, в котором хранится объект.  
Метод isEmpty() проверяет, находится ли объект в коробке.  
Метод put() помещает объект в коробку, но только если она пуста. Если в коробке уже есть объект, выбрасывается IllegalStateException.  
Метод take() возвращает объект из коробки и очищает коробку, то есть после извлечения ссылка на объект обнуляется.

Для демонстрации работы wildcard создан отдельный метод, принимающий коробку вида `Box<? super Point3D>`. Такой тип позволяет передавать коробки, которые могут хранить объект Point3D или его родительский тип. Внутри метода создаётся случайная трёхмерная точка, которая затем помещается в коробку.

Таким образом в задании показывается использование: обобщённого класса; параметра типа T; wildcard `? super Point3D`; безопасной работы с объектом внутри контейнера.

Пример:
Создаётся коробка `Box<Object>`.
В неё с помощью отдельного метода добавляется случайная точка Point3D.
После этого объект выводится на экран, затем извлекается методом take(), и коробка снова становится пустой.

</details>

## Задание 3.1 - Функция преобразования (обобщённый метод map)
<details>
<summary>Подробнее об задании</summary>

Обобщённый метод map

Описание:
Реализован обобщённый метод map, который позволяет преобразовать элементы списка одного типа в элементы другого типа. Метод принимает список исходных элементов и функцию преобразования, которая применяется к каждому элементу списка.

Логика решения:
Метод объявлен с двумя параметрами типа: `<T, P>`, где:  
T — тип элементов исходного списка  
P — тип элементов результирующего списка

Метод принимает:  
список `List<T>` с исходными элементами  
функцию `Function<T, P>`, которая преобразует элемент типа T в элемент типа P

Внутри метода создаётся новый список `ArrayList<P>`. Затем каждый элемент исходного списка проходит через функцию преобразования, и результат добавляется в новый список.  
Таким образом формируется новый список, содержащий преобразованные значения.

Пример:
Список строк: ["qwerty", "asdfg", "zx"]

При использовании функции `String::length` метод map возвращает список длин строк: [6, 5, 2]

</details>

## Задание 3.2 - Фильтрация элементов (обобщённый метод filter)
<details>
<summary>Подробнее об задании</summary>

Обобщённый метод filter

Описание:    
Реализован обобщённый метод filter, который позволяет отфильтровать элементы списка по заданному условию. Метод возвращает новый список, содержащий только те элементы, которые удовлетворяют указанному условию.

Логика решения:   
Метод объявлен с параметром типа `<T>`, где T — тип элементов списка.

Метод принимает:   
список `List<T>` с исходными элементами  
условие `Predicate<T>`, которое проверяет, должен ли элемент остаться в списке  

Внутри метода создаётся новый список `ArrayList<T>`. Далее каждый элемент исходного списка проверяется с помощью Predicate. Если условие возвращает true, элемент добавляется в новый список.  
Таким образом формируется список, содержащий только элементы, которые удовлетворяют заданному условию.

Пример:
Список чисел: [1, -3, 7, 0, -2]

При использовании условия `x -> x <= 0` метод filter возвращает: [-3, 0, -2]

В результате из исходного списка остаются только неположительные числа.
Таким образом каждый элемент исходного списка был преобразован в новое значение.

</details>

## Задание 3.3 - **Сокращение списка** (обобщённый метод reduce) 
<details>
<summary>Подробнее об задании</summary>
Обобщённый метод reduce

Описание:
Реализован обобщённый метод reduce, который позволяет объединить все элементы списка в одно итоговое значение. Метод последовательно применяет заданную операцию к элементам списка и возвращает результат вычисления.

Логика решения:
Метод объявлен с параметром типа <T>, где T — тип элементов списка.

Метод принимает:   
список List<T> с исходными элементами
функцию BinaryOperator<T>, которая объединяет два элемента списка
значение по умолчанию, которое возвращается, если список пуст

Если список пуст, метод сразу возвращает значение по умолчанию.
Если список содержит элементы, первый элемент принимается как начальное значение. Далее функция объединения применяется последовательно ко всем элементам списка.

Таким образом все элементы списка постепенно объединяются в одно итоговое значение.

Пример:
Список чисел: [1, -3, 7]

При использовании операции сложения Integer::sum метод reduce возвращает:5

Также метод корректно работает с пустым списком и возвращает значение по умолчанию, не вызывая ошибок выполнения.

</details>

## Задание 3.4 - **Коллекционирование элементов** (обобщённый метод collect)  
<details>
<summary>Подробнее об задании</summary>
  
Обобщённый метод collect

Описание:
Реализован обобщённый метод collect, который позволяет собрать элементы списка в новую структуру данных. Метод создаёт коллекцию указанного типа и добавляет в неё элементы списка с помощью специального аккумулятора.

Логика решения:
Метод объявлен с двумя параметрами типа <T, P>, где:
T — тип элементов исходного списка
P — тип результирующей коллекции

Метод принимает:
список List<T> с исходными элементами
Supplier<P>, который создаёт новую коллекцию
BiConsumer<P, T>, который добавляет элемент в коллекцию
Сначала создаётся новая коллекция с помощью Supplier. Далее каждый элемент исходного списка передаётся в аккумулятор BiConsumer, который добавляет элемент в результирующую структуру.

Таким образом метод позволяет гибко создавать различные коллекции, например List, Set или Map.

Пример:
Список строк: ["qwerty", "asdfg", "qwerty", "qw"]

При использовании коллекции типа Set метод collect возвращает набор уникальных элементов:

["qwerty", "asdfg", "qw"]

Таким образом повторяющиеся элементы автоматически исключаются из результата.

</details>

## Использование обобщённых типов (Generics)

Описание:   
В данной лабораторной работе были использованы возможности языка Java для работы с обобщёнными типами (Generics). Обобщения позволяют писать универсальный код, который может работать с различными типами данных без потери типобезопасности.

**Использованные возможности Generics:**

В ходе выполнения лабораторной работы были применены следующие элементы обобщённого программирования:

Обобщённые типы (Generic types)
Создан класс Box<T>, который может хранить объект любого типа. Тип объекта задаётся при создании коробки.

Типовые переменные и параметризация <T>
В классах и методах используются параметры типа (T, P), позволяющие работать с различными типами данных.

Маски типов (wildcards)
Использована маска типа ? super Point3D, позволяющая передавать в метод коробки, которые могут хранить объекты типа Point3D или его родительских типов.

Обобщённые методы
Реализованы методы map, filter, reduce и collect, которые могут работать со списками различных типов данных.

Автовывод типа (Type inference)
Компилятор автоматически определяет типы параметров при вызове обобщённых методов, что упрощает использование generics и делает код более читаемым.
